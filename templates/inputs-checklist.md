# Inputs Checklist

Gather these before running the kit. Keep personal copies under `inputs/` (gitignored)
so you never commit private data.

## For résumé work (review / improve / build / tailor)

- [ ] **Current résumé** (PDF or DOCX) — skip if using `resume-build` from scratch.
- [ ] **Target job description(s)** — required for `resume-tailor`, recommended for
      everything else. Paste full text into `inputs/target-jds.md`.
- [ ] **Metrics scratchpad** — real numbers to fill `[METRIC NEEDED]` gaps (user counts,
      % improvements, $ saved, team sizes, scale). Sources: performance reviews,
      dashboards, launch docs, Slack threads.

## For LinkedIn work (linkedin-rewrite)

- [ ] **LinkedIn profile PDF** — Profile → Resources → Save to PDF → download.
- [ ] **3–5 target job descriptions** (same file as above is fine).
- [ ] *(Optional)* **Voice doc** — fill `voice-doc-template.md` so output sounds like you.

## Target JDs file format

```
## JD 1 — <Role title> @ <Company>
<paste full job description>

## JD 2 — <Role title> @ <Company>
<paste full job description>
...
```

The more complete the JD text, the better the keyword and vocabulary matching.
